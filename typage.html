Data Typing

In C, there are different data types. Here are the most important ones:

char (a single character) / int (an integer) / short (a short integer) / long (a long integer) / float (a real number) / double (a double-precision real number) / void * (a pointer to data, the same as int *, char *, etc — see pointer page).

Each piece of data is represented in binary inside your computer — meaning using only 0 and 1. This is why we can do the following: 

#include <stdio.h>

int main() {
    printf("We can write the char '0' using the number 48 thanks to the ASCII table: %c.\n", 48);
    printf("We can also write the char 'A' using the number 65: %c.\n", 65);
    printf("It also works the other way around: the number 48 is represented by the char '%d'.\n", '0');
    return 0;
}

You might wonder: what’s the difference between all these data types?

The only difference is the number of bytes they take in your computer’s memory.

One byte is made up of 8 bits (0 or 1).

A char doesn’t need to store large numbers, since the ASCII table only maps numbers from 0 to 127 to characters.

Therefore, a char only uses 1 byte of memory, because 128 = 2^7. Since a byte contains 8 bits, we can store up to 2^8 possible values.

This means a char can hold numbers from -127 to 128 (the first bit indicates the sign, while the other 7 bits represent the value).
